🟠 SECTION 4 — HIGH: Typography Not Fluid
Mobile
h1 = 3.5rem fixed, h2 = 2.5rem fixed — overflow and layout breaks on phones
Only .hero-title uses clamp(). Every other heading uses fixed rem values. On a 375px iPhone, h1 is ~56px — it overflows narrow containers and creates uncomfortable reading. Section headings across all pages are affected.
css/style.css
❌ Fixed sizes:
h1 { font-size: 3.5rem; }  /* 56px — too large on mobile */
h2 { font-size: 2.5rem; }  /* 40px — still large */
h3 { font-size: 1.75rem; }
✅ Replace with fluid clamp() typography:
h1 { font-size: clamp(2rem, 5vw, 3.5rem); }
/* 32px on small screens → scales → 56px max on desktop */

h2 { font-size: clamp(1.6rem, 4vw, 2.5rem); }
/* 26px → 40px */

h3 { font-size: clamp(1.25rem, 3vw, 1.75rem); }
/* 20px → 28px */

/* Also make spacing fluid */
:root {
  --space-xl: clamp(48px, 8vw, 96px);
  --space-lg: clamp(28px, 5vw, 48px);
}
Mobile
Arabic text fixed at 1.4em with line-height 2.2 — causes unexpected section height blowout
Multi-line Arabic passages with these fixed values create very tall blocks on small screens, pushing content far below expectations. Bismillah at 2rem can also overflow on tiny screens.
✅ Make Arabic text responsive:
.arabic-text {
  font-size: clamp(1.1rem, 3vw, 1.4em);
  line-height: 2.2;
}
.bismillah {
  font-size: clamp(1.4rem, 5vw, 2rem);
}
@media (max-width: 480px) {

🟠 SECTION 6 — HIGH: Prayer Times JS Bugs
Memory Leak
setInterval() never cleared — accumulates on long sessions, doesn't detect day change
The 1-second countdown interval ID is never stored or cleared. After midnight, it continues showing wrong prayer times (previous day's data) until the user manually refreshes. On long visits, multiple intervals could stack.
js/prayer-times.js
❌ Current — no cleanup, no day detection:
function startCountdownTracker(timings) {
  setInterval(() => { ... }, 1000);  /* ID discarded, never cleared */
}
✅ Fix with proper interval management + midnight reset:
let countdownInterval = null;

function startCountdownTracker(timings) {
  if (countdownInterval) clearInterval(countdownInterval);  // Clear old

  countdownInterval = setInterval(() => {
    const now = new Date();

    // Auto-reset if day has changed
    const todayStr = now.toISOString().split('T')[0];
    if (localStorage.getItem('prayer_date') !== todayStr) {
      clearInterval(countdownInterval);
      fetchPrayerTimes();  // Re-fetch for new day
      return;
    }
    localStorage.setItem('prayer_date', todayStr);

    // ... rest of countdown logic unchanged ...
  }, 1000);
}

window.addEventListener('beforeunload', () => {
  if (countdownInterval) clearInterval(countdownInterval);
});

Prayer times show raw "Loading prayer times..." text — no loading skeleton
The most prominent homepage section shows plain loading text during the API call. On slow mobile connections (common in mosque settings with many concurrent users), this can last 2-4 seconds.

// In prayer-times.js — replace initial innerHTML:

}
Safety

The JS calculates Iqamah by adding 20 minutes to Adhan (5 for Maghrib). These are arbitrary guesses presented as the actual Masjid schedule. If a community member arrives based on this, they may find prayer has started or not yet begun.
